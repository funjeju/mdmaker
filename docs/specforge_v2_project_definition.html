SPECFORGE V2: ENGINE PROTOCOL

SpecForge V2

단일 문서 생성기에서 '프로덕트 지식 그래프 빌더(Product Knowledge Graph Builder)'로의 대진화. 피처 단위로 가변적 산출물을 지능적으로 오케스트레이션하고 변경에 따른 영향도를 눈으로 추적합니다.

📦 완성된 기획서(V2) 패키지를 손쉽게 오프라인으로 보관하고 실행하세요.
알림 메시지
01 // Paradigm Shift

문서 파편화의 종말과 지식 그래프의 도입

기존의 AI 스펙 도구들은 단순 텍스트 묶음(PRD, API, DB)을 개별 파일 단위로 내보냅니다. 하지만 실무에서 프로덕트의 빌드 단위는 파일이 아니라 '기능(Feature)'입니다. 기능이 확장되거나 테이블이 변경될 때 발생하는 문서 간 싱크 붕괴는 개발 프로세스의 심각한 마찰(Friction)을 초래합니다.

The Great Conversion

기존 V1 (File-Centric)

고정된 8종 MD 문서를 순차적으로 생성합니다. 하나의 명세(예: DB 필드)가 변경되면 관련 API, 테스트 코드, 개발 가이드를 유저가 일일이 수동으로 찾아다니며 수정해야 합니다.

신규 V2 (Feature & Graph-Centric)

프로덕트의 '기능(Feature)' 노드 중심 아키텍처로 개편됩니다. 하나의 데이터 구조나 비즈니스 룰이 변경되는 즉시, 수정이 필요한 연관 노드들을 그래프 상에서 실시간으로 시각화하고 동기화할 타겟을 직접 안내합니다.

02 // Core Killer Features

V2 핵심 기능 스펙 (3대 기둥)

📂

기능(Feature) 중심 지식 정렬

'회원가입', '결제' 등 실제 비즈니스 기능 단위로 폴더 및 노드가 생성되며, 해당 기능에 수반되는 PRD, UserFlow, API, DB, Test Case, Cursor 규칙이 수직 구조로 정렬·결합합니다.

📈

가변형 산출물 최적화 (Adaptive Calibration)

더 이상 8종 문서에 구속되지 않습니다. 기능 난이도(Spark ~ Enterprise)에 따라 AI가 판단하여 간단한 UI는 2개의 가벼운 명세서만, 결제 연동은 10개 이상의 다차원 문서를 실시간 오케스트레이션합니다.

⚡

시각적 영향도 분석 (Visual Impact Tracker)

개발과 설계의 최대 페인 포인트인 '수정 사이드 이펙트'를 제어합니다. 특정 속성(예: DB Schema) 수정 시 지식 그래프가 연결 관계를 추적하여 영향받는 노드(API, Test Case)에 시각적 알림을 보냅니다.

05 // V2 System Architecture

SpecForge V2 기술 아키텍처 레이어

지식 그래프와 영향도 분석 시스템을 지탱하기 위해 최신 Graph DB 패턴 및 데이터 관계 오케스트레이션 엔진이 백엔드 및 UI 구조에 장착됩니다.

레이어 주요 컴포넌트 역할 및 상세 기여
UI Layer (프론트엔드) Interactive Graph View
Feature Drawer 		- 텍스트 탭을 대체하는 노드-엣지 시각화 인터페이스
- 기능 폴더별 하위 산출물 가변 생성 UI 맵핑
- 영향도 분석에 기반한 노드별 경고 하이라이팅 및 승인 버블링
AI Engine Layer (분석 엔진) Impact Analysis Engine
Cross-Reference LLM Linker 		- 마크다운 분석을 통한 속성 데이터 추출 및 그래프 의존성 연산
- DB 구조 변경 시 영향받는 API 코드라인 및 테스트 조건 식별
- AI 코딩 에이전트를 위한 컨텍스트 주입 압축 규칙 엔진
Database Layer (저장소) Postgres JSONB + Neo4j - 문서 속성 데이터와 기능 노드 간의 관계 구조 데이터베이스화
- 문서 수정 시 연관 엣지(Edge)의 일괄 수정 트리거 및 실시간 상태 동기화 관리
